4 PrAnaViz

4.1 Introduction

The aim of this tool is to visualize and analyze the prescription data to calculate total prescribed quantity of different APIs and to explore spatio-temporal trends of different APIs for wider use.

We have created, PrAnaViz, in a familiar dashboard layout with two tabs: (1) Targeted API, (2) Non-targeted API, to visualize total quantity of different APIs at CCG region with resolution to individual postcode.

4.2 Database connection string

  • The demo version of the in-built PrAnaViz enables users to input their database credentials to link the processed prescription dataset.
  • This can added by clicking the database icon in the right side of the navigation bar, 1 as in Figure: 4.1.
  • A drop down menu as in Figure: 4.2 will be generated to input the database connection strings such as, username, password and hostname

4.3 Targeted Approach

  • In this tab, user can input a list of APIs, and find out the total prescription quantity of each API in the selected year, at the selected CCG region, as in the Figure: 4.3.
  • This tab also visualize the total quantity of APIs prescribed by month, GP, Chemical form, and Medicinal form for a CCG region in a selected year, Figure: 4.4.

4.4 Non-targeted Approach

  • Total prescription quantity of an individual API at different postcode per month at a CCG region, can be rendered in this tab, as in the Figure: 4.5.
  • The calculated total prescription quantity of an individual API at postcode level helps to find the hotspots.
  • User can download data as .csv file and publication ready image .eps and .pdf files.
